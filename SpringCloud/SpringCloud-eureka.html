注册中心Eureka

Eureka是Netflix开源的一款提供服务注册和发现的产品，它提供了完整的Service Registry和Service Discovery实现。也是springcloud体系中最重要最核心的组件之一。

Eureka由两个组件组成：Eureka服务器和Eureka客户端。Eureka服务器用作服务注册服务器。Eureka客户端是一个java客户端，用来简化与服务器的交互、作为轮询负载均衡器，并提供服务的故障切换支持。

 

Eureka的基本架构，由3个角色组成：

1、Eureka Server

  • 提供服务注册和发现

2、Service Provider

  • 服务提供方
  • 将自身服务注册到Eureka，从而使服务消费方能够找到

3、Service Consumer

  • 服务消费方
  • 从Eureka获取注册服务列表，从而能够消费服务

 

一、注册中心搭建

 

1、创建一个SpringBoot项目，加入依赖

这是eureka的服务端的依赖，也可以通过idea等工具在创建项目时勾选相关依赖选项

 <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-netflix-eureka-server</artifactId>
        </dependency>

 

2、配置文件

这是注册中心的配置

xxxxxxxxxx
#服务名
spring.application.name=server
server.port=8008
eureka.instance.hostname=localhost
​
#防止服务端把自己也注册
eureka.client.registerWithEureka= false
eureka.client.fetchRegistry=false
​
#注册中心地址
eureka.client.serviceUrl.defaultZone=http://${eureka.instance.hostname}:${server.port}/eureka/
​
​

 

3、启动类

启动类加上@EnableEurekaServer，声明为服务端

    }

三、消费者(ribbon)

在微服务架构中，业务都会被拆分成一个独立的服务，服务与服务的通讯是基于http restful的。Spring cloud有两种服务调用方式，一种是ribbon+restTemplate，另一种是feign。

 

我们先看ribbon，eureka客户端既可以作为消费者也可以作为提供者，所以其配置大同小异。

1、创建项目、导入依赖、配置文件

ribbon作为消费者，其依赖和提供者一样。

配置文件也类似。

xxxxxxxxxx
spring.application.name=ribbon
server.port=8010
eureka.client.serviceUrl.defaultZone=http://localhost:8008/eureka/
​

 

2、启动类配置

开启消费者注解，配置rest，启用负载均衡。

x
@SpringBootApplication
@EnableEurekaClient
@EnableDiscoveryClient
public class RibbonApplication {
​
    public static void main(String[] args) {
        SpringApplication.run(RibbonApplication.class, args);
    }
​
    @Bean
    @LoadBalanced
    RestTemplate restTemplate() {
        return new RestTemplate();
    }
​
}

 

3、启用远程调用

通过restTemplate.getForObject（）方法远程调用提供者的服务，参数1为服务提供者url，参数2为返回类型的class.

xxxxxxxxxx
@RestController
public class BuyTicketController {
​
​
    @Autowired
    RestTemplate restTemplate;
    
    @GetMapping(value = "/hello")
    public String buyTicket(@RequestParam String name) {
        return restTemplate.getForObject("http://PROVIDER/hello?name="+name,String.class);
    }
    
    @GetMapping(value = "/buy")
    public String buyTicket() {
        return restTemplate.getForObject("http://PROVIDER/buy",String.class);
    }
​
}

三、消费者(feign)

Feign是一个声明式的伪Http客户端，它使得写Http客户端变得更简单。使用Feign，只需要创建一个接口并注解。它具有可插拔的注解特性，可使用Feign 注解和JAX-RS注解。Feign支持可插拔的编码器和解码器。Feign默认集成了Ribbon，并和Eureka结合，默认实现了负载均衡的效果。

简而言之：

Feign 采用的是基于接口的注解 Feign 整合了ribbon，具有负载均衡的能力 整合了Hystrix，具有熔断的能力

1、创建项目、加入依赖、配置

            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-netflix-eureka-client</artifactId>
        </dependency>
        <dependency>

 

spring.application.name=feign
server.port=8011

2、提供一个接口

定义一个feign接口，通过@ FeignClient（“服务名”），来指定调用哪个服务以及对应的路径。

比如在代码中调用了provider服务的“/buy”接口，代码如下：

x
@FeignClient(value = "provider")
public interface BuyTicketIm {
​
    @RequestMapping("/hello")
    public String Hello(@RequestParam String name);
    @RequestMapping("/buy")
    public String buyTicket();
​
}

 

 

3、调用服务

在web层把接口加入控制器，此时可以访问了。

@RestController
public class BuyTicketController {
    @Autowired
    BuyTicketIm buyTicketIm;
    @GetMapping("/hello")
    public String Hello(@RequestParam String name){
        return buyTicketIm.Hello(name);
    }
    @GetMapping("/buy")
    public String buyTicket(){
        return buyTicketIm.buyTicket();
    }
}
